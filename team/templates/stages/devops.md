Objective:
Prepare release, deployment safety, and rollback readiness.

Inputs:
- Final code state and SRE/security guidance.

Outputs:
- Deployment plan and release checklist.
- Rollback and incident response notes.

Acceptance Criteria:
- Release process includes verification gates.
- Rollback path is clear and testable.
- Final report marks go/no-go with rationale.

Handoff:
Provide final release recommendation to orchestrator/user.
