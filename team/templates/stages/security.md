Objective:
Assess security posture and abuse resistance.

Inputs:
- QA report, architecture notes, and changed surfaces.

Outputs:
- Threat model delta.
- Actionable security findings and mitigations.

Acceptance Criteria:
- Covers auth, data handling, and abuse vectors.
- Identifies critical and high severity issues explicitly.
- Provides handoff to sre and devops.

Handoff:
Pass security controls and unresolved risks to sre/devops.
